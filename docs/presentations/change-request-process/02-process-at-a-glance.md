# Process at a Glance - RDK-M Core Contributors

Direct branching - core contributors have write access to the repository.

```
 Requirement     Ticket        Branch        PR & Review       Merge
 ──────────► ──────────► ──────────────► ──────────────► ──────────────►
                                          If major change:
  Stakeholder   GitHub       Feature        14-Day Review   Develop
  identifies    Issue        branch per     + 5-Day         branch
  need          created      ticket         Resolution      (protected)
```

**Every change is tracked, reviewed, and approved before it reaches the protected develop branch.**

See: [RDK-M Core Contributors: Direct Branching](https://rdkcentral.github.io/rdk-halif-aidl/0.13.0/standards/direct_branching/)
