# Step 4: PR Review Process

## Standard Changes

- PR reviewed by the `rdk-halif-aidl-pr-review-team` via CODEOWNERS
- Standard GitHub PR review and approval process applies

## Major Changes - 14+5 Review Cycle

A **major change** is a functionality change from the original design (e.g. new interfaces, modified API contracts, breaking changes).

**Phase A: 14-Day Open Review Window**

- PR assigned to **all mandatory and domain reviewers**
- All technical feedback within **14 calendar days**
- **Silence equals consent**

**Phase B: 5-Business-Day Resolution Sprint**

| Day | Activity                                                     |
|-----|--------------------------------------------------------------|
| 1-3 | **Implementation** - HAL Lead triages and addresses feedback |
| 4   | **Sanity Check** - 24-hour verification, no new features     |
| 5   | **The Merge** - Status updated to GREEN, merged to develop   |
