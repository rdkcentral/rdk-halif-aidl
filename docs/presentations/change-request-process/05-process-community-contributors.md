# Process at a Glance - Community Contributors

Fork-based branching - external contributors without direct write access.

```
 Fork Repo     Clone & Branch    Commit & Push     Submit PR         Review & Merge
 ──────────► ──────────────► ──────────────► ──────────────► ──────────────►
  Fork the      Clone locally    Push changes      PR submitted      Core Development
  repository    & create a       to your fork      to the original   Team reviews &
  on GitHub     topic branch     on GitHub         repository        approves
```

**The same governance, review, and approval process applies to all contributions regardless of source.**

See: [Community Contributors: Fork-Based Branching](https://rdkcentral.github.io/rdk-halif-aidl/0.13.0/standards/forked_based_branching/)
