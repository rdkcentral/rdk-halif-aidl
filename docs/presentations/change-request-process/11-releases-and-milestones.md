# Releases, Milestones & Changelogs

## Milestones

- Every ticket is **assigned to a milestone** corresponding to a target release version
- Milestones provide visibility on progress towards each release
- Enables tracking of open vs closed issues per release cycle

## Tags & GitHub Releases

- Each release is **tagged** in the git repository (e.g. `0.14.0`)
- A **GitHub Release** is published for every tag with auto-generated **release notes**
- Release notes summarise all changes, contributors, and linked PRs included in the release

Example: [Release 0.14.0](https://github.com/rdkcentral/rdk-halif-aidl/releases/tag/0.14.0)

## Changelog

- A **changelog is generated for every release**, providing a complete record of what changed between versions
- Combined with semantic versioning, this gives full traceability from requirement through to release
