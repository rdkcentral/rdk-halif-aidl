# Step 2: Development on Feature Branch

- Developer creates a **named branch** linked to the ticket: `feature/{issue#}_{synopsis}`
- Follows the **Git-Flow** branching model, consistent with internal git repositories
- First-time contributors must sign the **Contributor License Agreement (CLA)**
- Work is isolated - **no direct commits to develop or main**
- Branch naming enforced by **Git-Flow Branch Rules** ruleset
